# iOS 客户端：AI 本地智能路线规划（美团黑客松赛道五）

## 1. 项目目标
开发一个 iOS 客户端 App，为用户提供“多人从不同起点出发，智能推荐会合吃喝玩乐路线”的核心体验。整体风格采用 Liquid Glass（液态玻璃）设计语言，交互流畅、视觉高端。App 与后端 AI 路线规划服务协同，实现自然语言输入、多方案对比、实时动态调整路线。

## 2. 技术栈与要求
- **语言**：Swift 5.9+
- **UI 框架**：SwiftUI（iOS 17+），充分利用 `Glassmorphism` 视觉效果、`PhaseAnimator`、`KeyframeAnimator` 等新特性
- **最低部署目标**：iOS 17.0
- **架构**：MVVM + 协调器
- **网络层**：async/await + URLSession 调用后端 REST API
- **地图**：MapKit 集成，展示 POI 标记、路线覆盖
- **依赖管理**：Swift Package Manager（无第三方 UI 库，纯原生实现 Liquid Glass）
- **数据持久化**：`@AppStorage` 存储用户偏好，`SwiftData` 存储历史路线

## 3. 核心功能需求

### 3.1 多用户不同起点规划（情侣模式）
- 用户可为每个参与者设置名称、头像颜色和出发位置（支持手动搜索地址或地图长按选点）。
- 界面提供“添加好友”按钮，快速切换单人/双人/多人规划模式。
- 双人模式下，用户 A 和用户 B 分别设置起点后，系统自动寻找“中间公平会合点”，并围绕会合点推荐餐饮、娱乐路线。
- 路线方案中标注：每个人的出发时间、通勤方式（公交/驾车/步行）、预计到达会合点时间，以及共同游玩的完整路线。

### 3.2 自然语言输入
- 主界面底部始终悬浮一个毛玻璃输入栏，提示文本“比如：想吃火锅再去看展，预算 300，不排队”。
- 支持语音输入（Speech Recognizer）转换为文本。
- 输入发送至后端 `/api/route/plan`，展示加载动画（液态流动效果）。

### 3.3 路线方案展示
- 后端返回多个方案后，以卡片流（横向分页）展示。每张卡片用 Liquid Glass 风格呈现：
  - 左上角方案标签（体验优先/最省时/最省钱）
  - 路线总时长、总花费、POI 串联简图
  - 每个 POI 的缩略信息（名称、评分、排队预估）
  - 右侧滑动查看详细推荐理由（基于后端返回的自然语言解释）。
- 地图视图与卡片联动：选中卡片时，地图自动缩放至路线全景并高亮路径。

### 3.4 动态实时调整
- 在路线详情页提供自然语言调整输入框：“换一家不用排队的咖啡店”。
- 调整请求调用 `/api/route/adjust`，响应后局部刷新卡片和地图，保留原有会话状态。
- 支持撤销/重做最后调整（基于本地状态快照）。

### 3.5 多人协同细节
- 生成路线后，每个参与者在自己设备上同样可以看到路线，并实时显示好友的当前位置（使用模拟位置，黑客松可硬编码移动动画）。
- 在路线时间线中，用不同颜色标注每位成员的独立行程段（出发→会合）和共同行程段。

## 4. Liquid Glass 设计系统

请使用 SwiftUI 自定义 Modifier 实现以下全局风格：

- **毛玻璃背景**：`ZStack` 中放置 `Color.clear.background(.ultraThinMaterial)`，叠加彩色噪点渐变（径向渐变 `RadialGradient` 模拟玻璃边缘高光）。
- **液态动效**：所有容器（卡片、按钮、输入框）的圆角使用 `RoundedRectangle(cornerRadius: 30, style: .continuous)`，边缘带有动态光晕（`shadow(color: .white.opacity(0.3), radius: 10)`）。
- **按钮与控件**：按钮按下时呈现“水波扩散”效果（使用 `Circle` + `scaleEffect` 动画）。输入框聚焦时边框渐变为流动的彩虹色（`AngularGradient` 旋转动画）。
- **颜色方案**：主色调为透明紫蓝渐变，卡片底色为 `#FFFFFF` 10% 透明度，文字使用系统 `primary` 和 `secondary` 色以适应深色/浅色模式。
- **字体**：大标题使用 `SF Pro Rounded`，正文使用 `SF Pro Text`，营造友好圆润感。
- **页面转场**：所有导航采用 `matchedGeometryEffect` 实现 Hero 动画（如首页卡片点击展开至详情页时，图片和标题平滑过渡）。

## 5. 界面与交互流程

### 5.1 首页（发现模式）
- 顶部：动态问候语（根据时间显示“下午好”），右侧好友头像切换多用户模式。
- 中部：地图占据上半屏，默认显示用户当前位置。地图上以气泡形式弹出附近热门 POI。
- 底部：输入栏 + 浮动按钮“路线规划”和“发起多人”。
- 点击“发起多人”后弹出底部表单：添加好友（头像、昵称、起点选取），起点可在小地图上拖动图钉选取。

### 5.2 路线生成中
- 全屏液态玻璃转场，中心为流动的半透明环（旋转的 Gradient 环），显示“正在为你计算最优路线...”。
- 支持取消按钮。

### 5.3 路线结果页
- 顶部地图：标注各个 POI，会合点用特殊心形/星形标记，不同用户的起点用各自头像标记，路径用渐变色线条绘制。
- 下方横向卡片滚动 `ScrollView(.horizontal)`，每张卡片为独立的路线方案。
- 点击卡片展开详细日程：时间线列表，每个步骤附带图标、时长、说明。
- 滑动到底部可输入自然语言调整。

### 5.4 导航模拟
- 提供“开始导航”按钮，进入模拟导航模式：使用 MapKit 的 `MKRoute` 展示方向，模拟用户位置沿路线移动（使用定时器更新 `CLLocationCoordinate2D`）。

## 6. 后端接口对接
